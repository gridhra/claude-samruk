# ソース品質・確信度・評価の共通規約

全スキル・全ワーカーが従う定義。**この文書が唯一の定義箇所**であり、他所に再定義しない。

## ソース品質（A〜E）

| 評価 | 定義 | 例 |
|---|---|---|
| A | 一次資料・公式資料 | 原典・書簡・判決文・公式機関の一次発表・原論文 |
| B | 定評ある二次資料 | 査読論文・学術書・専門事典・標準的な注釈書 |
| C | 信頼できる一般資料 | 大手報道・専門メディア・研究者による一般向け解説 |
| D | 出典の不明瞭な資料 | 個人ブログ・まとめ記事・出典を示さない解説 |
| E | 不確実・要検証 | 真偽不明・利害関係が疑われる・機械生成の疑い |

- D・E は単独では典拠にしない。裏取りの端緒としてのみ使う。
- 翻訳・孫引きは原典より1段階低く評価する。

## 確信度（高・中・低）

| 確信度 | 基準 |
|---|---|
| 高 | A/B級の**独立した**複数ソースが一致 |
| 中 | A/B級1件のみ、またはC級複数の一致 |
| 低 | C級以下のみ、またはソース間に矛盾がある |

- 独立性: 同一の元ネタに由来するソース群は1件と数える。

## 報告の構造

- **BLUF**: 結論を冒頭3〜5行で述べる。
- **構造化サマリ**（ワーカー→司令塔の返却形式）:
  1. 結論（担当範囲に対する答え）
  2. 典拠（URLまたは文献名 + 発行日/取得日）
  3. 各ソースのA〜E評価
  4. 確信度（高・中・低）と根拠
  5. 限界・未解決点

## 5軸評価（LLM-as-judge）

verifier が報告書を各軸1〜5で採点する。**全軸4以上**で合格。

| 軸 | 問い |
|---|---|
| 事実正確性 | 記載された事実は典拠と整合しているか |
| 引用正確性 | 引用・要約は原典の趣旨を歪めていないか（抜き取り検証を含む） |
| 網羅性 | 問いに対する主要な論点・対立見解を落としていないか |
| ソース品質 | A/B級中心か。D/E級に依存していないか |
| ツール効率 | 調査規模は問いの複雑度に見合っていたか（過剰・過少とも減点） |

不合格の場合、verifier は未達の軸と差し戻し先フェーズを指定する。
